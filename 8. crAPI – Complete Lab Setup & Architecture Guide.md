## 🛠 Installing Prerequisites (Kali / Debian)

### 1️⃣ Update Your System
```bash
apt update
```

### 2️⃣ Install Docker & Docker Compose
```bash
apt install docker.io docker-compose -y
```

### 3️⃣ Enable and Start the Docker Service
```bash
systemctl enable docker
systemctl start docker
```

### 4️⃣ Move Into Your Working Directory
```bash
cd /opt
```

---

## 🚗 crAPI Setup — ZIP Download Method

### 📦 Grab the crAPI Source
```bash
curl -L -o /tmp/crapi.zip https://github.com/OWASP/crAPI/archive/refs/heads/main.zip
```

### 📂 Unzip It
```bash
unzip /tmp/crapi.zip
```

### 📁 Move Into the Docker Deployment Folder
```bash
cd crAPI-main/deploy/docker
```

### ⬇️ Pull the Required Images
```bash
docker-compose pull
```

### 🚀 Bring crAPI Up (Listening on All Interfaces)
```bash
LISTEN_IP="0.0.0.0" docker-compose -f docker-compose.yml --compatibility up -d
```

## 🚗 Quick Setup — All-in-One Command Block (Recommended)

> ✅ Tested and working.

```bash
curl -L -o /tmp/crapi.zip https://github.com/OWASP/crAPI/archive/refs/heads/main.zip
unzip /tmp/crapi.zip
cd crAPI-main/deploy/docker
docker-compose pull
LISTEN_IP="0.0.0.0" docker-compose -f docker-compose.yml --compatibility up -d
```

## 🌐 Accessing the App

Once it's up, swap in your own host's IP wherever you see `<your-ip>` below — e.g. if your machine's IP is `192.168.1.103`, that's what goes in place of the placeholder.

### 🔗 Service Access Table

| Service            | URL                                              | Purpose          |
|---------------------|---------------------------------------------------|-------------------|
| 🌐 Web App (HTTP)  | `http://<your-ip>:8888/login`                    | Main UI           |
| 🔐 Web App (HTTPS) | `https://<your-ip>:8443/login`                   | TLS testing       |
| 📧 Mailhog UI       | `http://<your-ip>:8025`                          | Captured emails   |

---

## 🧠 Notes for Pentesters

- Use **HTTP (8888)** for general functional testing.
- Use **HTTPS (8443)** specifically when probing for TLS misconfigurations.
- Mailhog is handy for digging into:
  - Email verification flows
  - Password-reset attack paths
  - Token leakage in outgoing emails
