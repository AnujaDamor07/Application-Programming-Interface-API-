# 🛒 Vulnerable E-Commerce API — Lab Notes

🔗 **Repository:** [Vulnerable-Ecommerce-API](https://github.com/InfoSecWarrior/Offensive-Pentesting-Lab/tree/main/Vulnerable-Ecommerce-API)

> ⚠️ **Heads up:** this app is packed with intentional security flaws built specifically for training purposes. Don't deploy it anywhere public, and never feed it real data.

A realistic, production-style e-commerce API riddled with deliberate vulnerabilities — built as a hands-on playground for security training and pentesting practice.

---

## 🎯 What's Inside

This API mimics a mid-sized e-commerce platform, complete with:
- User authentication and authorization (JWT-based)
- Product catalog management
- Shopping cart functionality
- Order processing with simulated payments
- A review/rating system
- An admin panel
- File upload support
- Simulated internal microservices
- Webhook integrations
- A coupon/discount system

---

## 📋 What's Actually Done Securely

Not everything here is broken on purpose — some parts are intentionally solid, which makes the contrast more useful for learning:
- Passwords hashed with bcrypt
- JWT-based authentication
- Input validation on a subset of endpoints
- Proper RBAC enforced on some (not all) admin endpoints
- Parameterized SQL queries in most places — but not universally

## 🚀 Getting It Running

### Prerequisites
- Node.js 18+
- npm or yarn
- Docker (optional, if you'd rather containerize it)

### Checking Whether Node.js Is Already Installed

```bash
node -v
npm -v
```

If both come back with errors, neither is installed yet.

### Installing Node.js (npm Comes Bundled With It)

**Option 1 — Through Your Package Manager (quickest)**

For Ubuntu / Kali / Debian:
```bash
apt update
apt install nodejs npm
apt install python3-setuptools
```

**Option 2 — Via NVM (better if you need to juggle multiple Node versions)**

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```

Reload your shell config:
```bash
source ~/.zshrc
```

Then install Node:
```bash
nvm install node
```

**Verify it worked:**
```bash
node -v
npm -v
```

## 🛠️ Installing the Lab Itself

### Method 1 — Running It Locally

1. **Move into the project folder**
```bash
cd Vulnerable-Ecommerce-API
```

2. **Install dependencies**
```bash
npm install sqlite3@latest
npm install
```

3. **Set up your environment file** (the defaults are fine for testing)
```bash
cp .env.example .env
```

4. **Seed the database**
```bash
npm run seed
```

5. **Start the server**
```bash
npm start
```

6. **Access it here:**
- API base URL: `http://localhost:3000/api/v1`
- API docs: `http://localhost:3000/api-docs`
- Health check: `http://localhost:3000/health`

### Method 2 — Running It via Docker

1. **Pull and run the image**
```bash
docker run -p 3000:3000 infosecwarrior/vapi:latest
```

2. **Seed the database from another terminal**
```bash
docker-compose exec api npm run seed
```

3. **Access it at:** `http://localhost:3000`
