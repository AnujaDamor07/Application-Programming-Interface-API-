## 1️⃣ Installation

### 📂 Step 1: Move to Your Working Directory
```bash
cd /opt
```

### 📥 Step 2: Clone the Repo
```bash
git clone https://github.com/roottusk/vapi.git
```

### 📁 Step 3: Enter the Project Folder
```bash
cd /opt/vapi
```

### 🐳 Step 4: Spin Up the Lab via Docker Compose
```bash
docker-compose up -d
```

### ✅ Confirm Everything's Running
```bash
docker ps
netstat -nltup
```

## 3️⃣ What vAPI Actually Is

vAPI is a **deliberately minimal, vulnerable REST API**. It's noticeably lighter than crAPI, which makes it a great fit for:

- Practicing API enumeration
- Injection testing
- Auth-bypass testing
- Finding IDOR issues
- Basic fuzzing drills

Treat it as a **good warm-up lab before moving on to crAPI** in your overall pentesting workflow.

## 4️⃣ Initial Enumeration

### Step 1 — Directory Enumeration
```bash
ffuf -u http://<your-ip>/FUZZ -w /usr/share/wordlists/dirb/common.txt
```

Also try:
```bash
ffuf -u http://<your-ip>/vapi/FUZZ -w wordlist.txt
```

### Step 2 — HTTP Method Testing
```bash
curl -X OPTIONS http://<your-ip>/vapi/api1 -i
```

Run through:
- GET
- POST
- PUT
- DELETE

Insecure or unintended methods are sometimes left wide open.

### Step 3 — Parameter Discovery
```bash
ffuf -u "http://<your-ip>/vapi/api1?FUZZ=test" -w param.txt
```

Look out for:
- Hidden parameters
- Debug flags
- Admin-only flags

## 5️⃣ Common Vulnerabilities to Hunt For

### 1. SQL Injection

Try payloads like:
```text
' OR 1=1--
" OR "1"="1
```

```bash
sqlmap -u "http://<your-ip>/vapi/api1?id=1" --batch --dbs
```

### 2. Authentication Bypass

**Try:**
```text
admin' --
admin" OR "1"="1
```

**Also check for:**
- Hardcoded credentials
- Missing password validation
- JWT tampering (if JWTs are in play)

### 3. IDOR (Broken Object Level Authorization)

If you find an endpoint like:
```text
/vapi/api1/user/1
```

**Try:**
```text
/vapi/api1/user/2
/vapi/api1/user/3
```

**Check for:**
- Horizontal privilege escalation
- Whether you can pull another user's data

### 4. Input Validation Issues

**Test with:**
- Very long strings
- Special characters
- Tampered JSON
- Negative values
- Oversized numeric values

### 5. Mass Assignment

If the request body looks like:
```json
{
  "username": "rahul",
  "role": "admin"
}
```

**Try sneaking in extra fields like:**
- `isAdmin`
- `role`
- `balance`
- `user_id`

## 6️⃣ Recommended Burp Workflow

1. Route the request through Burp's proxy
2. Send it to Repeater
3. Start modifying:
   - ID values
   - The HTTP method
   - Headers
   - The JSON body
4. Compare how the responses differ

## 7️⃣ vAPI vs crAPI

| Feature         | vAPI    | crAPI    |
|-------------------|---------|----------|
| Complexity        | Low     | High     |
| Microservices     | No      | Yes      |
| JWT                | Limited | Heavy    |
| Business logic     | Basic   | Advanced |
| GraphQL            | No      | No       |

vAPI is best suited for:
- Basic API fuzzing drills
- Automation/tooling practice
- Getting comfortable with sqlmap
